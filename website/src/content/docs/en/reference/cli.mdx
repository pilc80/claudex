## Command Split

```
claudex [CLAUDE_ARGS...]
claudex-config [OPTIONS] [COMMAND]
```

`claudex` is the Claude Code launcher. It passes its arguments through to
Claude Code unchanged. Use environment variables for claudex runtime settings.

`claudex-config` owns setup and management commands.

| Option | Description |
|--------|-------------|
| `--config <PATH>` | Override config file path |
| `--version` | Print version |
| `--help` | Print help |

Use `CLAUDEX_CONFIG=<PATH>` when launching with `claudex`. Use
`claudex-config --config <PATH>` for management commands.

## Commands

### launch

Run Claude Code with a specific provider profile.

```bash
CLAUDEX_PROFILE=<PROFILE> claudex [PROMPT] [CLAUDE_ARGS...]
```

| Argument / Option | Description |
|----------|-------------|
| `CLAUDEX_PROFILE` | Profile name, or `auto` for smart routing |
| `PROMPT` | Optional prompt for non-interactive (one-shot) execution |
| `CLAUDEX_MODEL` | Override the default model for this session |
| `CLAUDEX_HYPERLINKS=on` | Force enable OSC 8 terminal hyperlinks |
| `--print` | Print the response and exit (non-interactive mode) |
| `--dangerously-skip-permissions` | Skip all permission prompts (use with `--print` for CI/CD) |
| `CLAUDE_ARGS...` | Arguments passed unchanged to `claude` |

```bash
# Use Grok (interactive)
CLAUDEX_PROFILE=grok claudex

# Use Grok with a specific model
CLAUDEX_PROFILE=grok CLAUDEX_MODEL=grok-3-mini-beta claudex

# Smart routing
CLAUDEX_PROFILE=auto claudex

# Force terminal hyperlinks
CLAUDEX_PROFILE=grok CLAUDEX_HYPERLINKS=on claudex

# Non-interactive one-shot execution
CLAUDEX_PROFILE=grok claudex "Explain this codebase" --print

# CI/CD pipeline usage (no prompts, no permissions)
CLAUDEX_PROFILE=grok claudex "Fix lint errors" --print --dangerously-skip-permissions

# Pass args to claude
CLAUDEX_PROFILE=grok claudex --verbose
```

### `profile`

Manage provider profiles.

#### `profile list`

```bash
claudex-config profile list
```

List all configured profiles with their status, provider type, and model.

#### `profile add`

```bash
claudex-config profile add
```

Interactive wizard to add a new profile. Guides through provider selection, API key entry (with optional keyring storage), model selection, and connectivity testing.

#### `profile show`

```bash
claudex-config profile show <NAME>
```

Show details for a specific profile including all configuration fields.

#### `profile remove`

```bash
claudex-config profile remove <NAME>
```

Remove a profile from the configuration.

#### `profile test`

```bash
claudex-config profile test <NAME|all>
```

Test connectivity for a specific profile or all profiles. Sends a lightweight request to the provider's `/models` endpoint.

### `auth`

Manage OAuth subscription authentication for provider profiles.

#### `auth login`

```bash
claudex-config auth login <PROVIDER> [OPTIONS]
```

| Option | Description |
|--------|-------------|
| `--profile <NAME>` | Associate with a specific profile |
| `--enterprise-url <URL>` | Custom enterprise server URL (for self-hosted GitLab, etc.) |
| `--headless` | Use headless (device code) flow, skip browser launch |

Log in to a provider using OAuth. Supported providers: `claude`, `openai`, `google`, `qwen`, `kimi`, `github`, `gitlab`.

```bash
# Log in to OpenAI (reads from Codex CLI)
claudex-config auth login openai --profile codex-sub

# Log in to Claude (reads from ~/.claude)
claudex-config auth login claude

# Log in to Google
claudex-config auth login google --profile gemini-sub

# Log in to GitLab with enterprise URL
claudex-config auth login gitlab --enterprise-url https://gitlab.mycompany.com

# Headless mode (no browser, device code only)
claudex-config auth login github --headless
```

#### `auth status`

```bash
claudex-config auth status [PROVIDER]
```

Show the current authentication status. Without a provider argument, shows status for all configured OAuth profiles.

```bash
# Check all OAuth profiles
claudex-config auth status

# Check a specific provider
claudex-config auth status openai
```

#### `auth refresh`

```bash
claudex-config auth refresh <PROVIDER>
```

Manually refresh the OAuth token for a provider. Tokens are also refreshed automatically by the proxy when they expire.

#### `auth logout`

```bash
claudex-config auth logout <PROVIDER>
```

Remove stored OAuth tokens for a provider from the system keyring.

### `proxy`

Manage the translation proxy server.

#### `proxy start`

```bash
claudex-config proxy start [OPTIONS]
```

| Flag | Description |
|------|-------------|
| `-p, --port PORT` | Override proxy port (default: from config) |
| `-d, --daemon` | Run as background daemon |

#### `proxy stop`

```bash
claudex-config proxy stop
```

Stop the proxy daemon (sends SIGTERM via PID file).

#### `proxy status`

```bash
claudex-config proxy status
```

Show whether the proxy is running and on which port.

### `dashboard`

```bash
claudex-config dashboard
```

Launch the TUI dashboard for real-time monitoring and quick-launch.

### `config`

Manage configuration files.

#### `config show`

```bash
claudex-config config show
```

Display the loaded config file path, search order, and current configuration values.

#### `config path`

```bash
claudex-config config path
```

Print only the resolved config file path.

#### `config init`

```bash
claudex-config config init
```

Create a new config file in the current directory from the built-in template.

#### `config recreate`

```bash
claudex-config config recreate
```

Recreate the config file from `config.example.toml`, overwriting the existing config.

#### `config edit`

```bash
claudex-config config edit
```

Open the config file in your `$EDITOR` (or `$VISUAL`).

#### `config validate`

```bash
claudex-config config validate
```

Validate config syntax, profile references (backup_providers, router rules), and OAuth provider settings. Reports errors and warnings.

#### `config get`

```bash
claudex-config config get <KEY>
```

Get a specific configuration value by key path (e.g., `proxy_port`, `profiles.0.name`).

#### `config set`

```bash
claudex-config config set <KEY> <VALUE>
```

Set a specific configuration value by key path.

#### `config export`

```bash
claudex-config config export
```

Export the current merged configuration to stdout in TOML format.

### `sets`

Manage configuration sets (installable bundles of rules, skills, and MCP servers).

#### `sets add`

```bash
claudex-config sets add <PATH>
```

Install a configuration set from a local directory containing a `manifest.json` (or `manifest.yaml`).

#### `sets remove`

```bash
claudex-config sets remove <NAME>
```

Remove an installed configuration set by name.

#### `sets list`

```bash
claudex-config sets list
```

List all installed configuration sets with their version, description, and component summary.

#### `sets update`

```bash
claudex-config sets update <NAME> <PATH>
```

Update an existing configuration set from a new manifest path.

#### `sets show`

```bash
claudex-config sets show <NAME>
```

Show detailed information about an installed configuration set, including all components.

### `update`

```bash
claudex-config update [OPTIONS]
```

| Flag | Description |
|------|-------------|
| `--check` | Only check for updates, don't install |

Without `--check`, downloads and installs the latest version from GitHub Releases.
