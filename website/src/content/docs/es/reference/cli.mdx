## Uso global

```
claudex [CLAUDE_ARGS...]
claudex-config [OPTIONS] [COMMAND]
```

`claudex` is the Claude-compatible launcher. `claudex-config` owns setup and management commands. Use `CLAUDEX_PROFILE`, `CLAUDEX_MODEL`, `CLAUDEX_HYPERLINKS`, and `CLAUDEX_CONFIG` for launch-time claudex settings.

| Opcion | Descripcion |
|--------|-------------|
| `--config <PATH>` | Sobreescribir la ruta del archivo de configuracion |
| `--version` | Mostrar version |
| `--help` | Mostrar ayuda |

Cuando se ejecuta sin un comando, Claudex lanza el panel TUI si hay perfiles configurados, o muestra un mensaje de bienvenida si no existen perfiles.

## Comandos

### launch

Ejecuta Claude Code con un perfil de proveedor especifico.

```bash
CLAUDEX_PROFILE=<PROFILE> claudex [PROMPT] [CLAUDE_ARGS...]
```

| Argumento / Opcion | Descripcion |
|--------------------|-------------|
| `CLAUDEX_PROFILE` | Nombre del perfil, o `auto` para enrutamiento inteligente |
| `PROMPT` | Prompt opcional para ejecucion no interactiva (un solo uso) |
| `CLAUDEX_MODEL` | Sobreescribir el modelo por defecto para esta sesion |
| `CLAUDEX_HYPERLINKS=on` | Forzar activacion de hipervinculos OSC 8 en terminal |
| `--print` | Imprimir la respuesta y salir (modo no interactivo) |
| `--dangerously-skip-permissions` | Omitir todas las solicitudes de permisos (usar con `--print` para CI/CD) |
| `ARGS...` | Argumentos adicionales pasados a `claude` (despues de `--`) |

```bash
# Usar Grok (interactivo)
CLAUDEX_PROFILE=grok claudex

# Usar Grok con un modelo especifico
CLAUDEX_PROFILE=grok CLAUDEX_MODEL=grok-3-mini-beta claudex

# Enrutamiento inteligente
CLAUDEX_PROFILE=auto claudex

# Forzar hipervinculos en terminal
CLAUDEX_PROFILE=grok CLAUDEX_HYPERLINKS=on claudex

# Ejecucion no interactiva de un solo uso
CLAUDEX_PROFILE=grok claudex "Explain this codebase" --print

# Uso en pipeline CI/CD (sin confirmaciones, sin permisos)
CLAUDEX_PROFILE=grok claudex "Fix lint errors" --print --dangerously-skip-permissions

# Pasar argumentos adicionales a claude
CLAUDEX_PROFILE=grok claudex --verbose
```

### `profile`

Gestiona los perfiles de proveedores.

#### `profile list`

```bash
claudex-config profile list
```

Lista todos los perfiles configurados con su estado, tipo de proveedor y modelo.

#### `profile add`

```bash
claudex-config profile add
```

Asistente interactivo para agregar un nuevo perfil. Guia por la seleccion del proveedor, la entrada de clave API (con almacenamiento opcional en el llavero), la seleccion del modelo y la prueba de conectividad.

#### `profile show`

```bash
claudex-config profile show <NAME>
```

Muestra los detalles de un perfil especifico incluyendo todos los campos de configuracion.

#### `profile remove`

```bash
claudex-config profile remove <NAME>
```

Elimina un perfil de la configuracion.

#### `profile test`

```bash
claudex-config profile test <NAME|all>
```

Prueba la conectividad de un perfil especifico o de todos los perfiles. Envia una solicitud ligera al endpoint `/models` del proveedor.

### `auth`

Gestiona la autenticacion OAuth por suscripcion para los perfiles de proveedores.

#### `auth login`

```bash
claudex-config auth login <PROVIDER> [OPTIONS]
```

| Opcion | Descripcion |
|--------|-------------|
| `--profile <NAME>` | Asociar con un perfil especifico |
| `--enterprise-url <URL>` | URL personalizada de servidor enterprise (para GitLab autogestionado, etc.) |
| `--headless` | Usar flujo headless (codigo de dispositivo), omitir lanzamiento del navegador |

Inicia sesion en un proveedor usando OAuth. Proveedores soportados: `claude`, `openai`, `google`, `qwen`, `kimi`, `github`, `gitlab`.

```bash
# Iniciar sesion en OpenAI (lee desde Codex CLI)
claudex-config auth login openai --profile codex-sub

# Iniciar sesion en Claude (lee desde ~/.claude)
claudex-config auth login claude

# Iniciar sesion en Google
claudex-config auth login google --profile gemini-sub

# Iniciar sesion en GitLab con URL enterprise
claudex-config auth login gitlab --enterprise-url https://gitlab.mycompany.com

# Modo headless (sin navegador, solo codigo de dispositivo)
claudex-config auth login github --headless
```

#### `auth status`

```bash
claudex-config auth status [PROVIDER]
```

Muestra el estado de autenticacion actual. Sin argumento de proveedor, muestra el estado de todos los perfiles OAuth configurados.

```bash
# Comprobar todos los perfiles OAuth
claudex-config auth status

# Comprobar un proveedor especifico
claudex-config auth status openai
```

#### `auth refresh`

```bash
claudex-config auth refresh <PROVIDER>
```

Refresca manualmente el token OAuth para un proveedor. Los tokens tambien se refrescan automaticamente por el proxy cuando expiran.

#### `auth logout`

```bash
claudex-config auth logout <PROVIDER>
```

Elimina los tokens OAuth almacenados para un proveedor del keyring del sistema.

### `proxy`

Gestiona el servidor proxy de traduccion.

#### `proxy start`

```bash
claudex-config proxy start [OPTIONS]
```

| Flag | Descripcion |
|------|-------------|
| `-p, --port PORT` | Sobreescribir el puerto del proxy (por defecto: desde la configuracion) |
| `-d, --daemon` | Ejecutar como daemon en segundo plano |

#### `proxy stop`

```bash
claudex-config proxy stop
```

Detiene el daemon del proxy (envia SIGTERM via archivo PID).

#### `proxy status`

```bash
claudex-config proxy status
```

Muestra si el proxy esta en ejecucion y en que puerto.

### `dashboard`

```bash
claudex-config dashboard
```

Lanza el panel TUI para monitoreo en tiempo real y lanzamiento rapido.

### `config`

Gestiona los archivos de configuracion.

#### `config show`

```bash
claudex-config config show
```

Muestra la ruta del archivo de configuracion cargado, el orden de busqueda y los valores de configuracion actuales.

#### `config path`

```bash
claudex-config config path
```

Muestra solo la ruta resuelta del archivo de configuracion.

#### `config init`

```bash
claudex-config config init
```

Crea un nuevo archivo de configuracion en el directorio actual a partir de la plantilla integrada.

#### `config recreate`

```bash
claudex-config config recreate
```

Recrea el archivo de configuracion desde `config.example.toml`, sobreescribiendo la configuracion existente.

#### `config edit`

```bash
claudex-config config edit
```

Abre el archivo de configuracion en tu `$EDITOR` (o `$VISUAL`).

#### `config validate`

```bash
claudex-config config validate
```

Valida la sintaxis de configuracion, las referencias de perfiles (backup_providers, reglas del router) y los ajustes de proveedores OAuth. Reporta errores y advertencias.

#### `config get`

```bash
claudex-config config get <KEY>
```

Obtiene un valor especifico de configuracion por ruta de clave (por ejemplo, `proxy_port`, `profiles.0.name`).

#### `config set`

```bash
claudex-config config set <KEY> <VALUE>
```

Establece un valor especifico de configuracion por ruta de clave.

#### `config export`

```bash
claudex-config config export
```

Exporta la configuracion combinada actual a stdout en formato TOML.

### `sets`

Gestiona los conjuntos de configuracion (paquetes instalables de reglas, habilidades y servidores MCP).

#### `sets add`

```bash
claudex-config sets add <PATH>
```

Instala un conjunto de configuracion desde un directorio local que contenga un `manifest.json` (o `manifest.yaml`).

#### `sets remove`

```bash
claudex-config sets remove <NAME>
```

Elimina un conjunto de configuracion instalado por nombre.

#### `sets list`

```bash
claudex-config sets list
```

Lista todos los conjuntos de configuracion instalados con su version, descripcion y resumen de componentes.

#### `sets update`

```bash
claudex-config sets update <NAME> <PATH>
```

Actualiza un conjunto de configuracion existente desde una nueva ruta de manifiesto.

#### `sets show`

```bash
claudex-config sets show <NAME>
```

Muestra informacion detallada sobre un conjunto de configuracion instalado, incluyendo todos los componentes.

### `update`

```bash
claudex-config update [OPTIONS]
```

| Flag | Descripcion |
|------|-------------|
| `--check` | Solo comprobar actualizaciones, no instalar |

Sin `--check`, descarga e instala la ultima version desde GitHub Releases.
