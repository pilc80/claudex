## グローバルオプション

```
claudex [CLAUDE_ARGS...]
claudex-config [OPTIONS] [COMMAND]
```

`claudex` is the Claude-compatible launcher. `claudex-config` owns setup and management commands. Use `CLAUDEX_PROFILE`, `CLAUDEX_MODEL`, `CLAUDEX_HYPERLINKS`, and `CLAUDEX_CONFIG` for launch-time claudex settings.

| オプション | 説明 |
|-----------|------|
| `--config <PATH>` | 設定ファイルパスを上書き |
| `--version` | バージョンを表示 |
| `--help` | ヘルプを表示 |

コマンドなしで実行すると、プロファイルが設定されている場合は TUI ダッシュボードを起動し、プロファイルが存在しない場合はウェルカムメッセージを表示します。

## コマンド

### launch

特定のプロバイダープロファイルで Claude Code を実行します。

```bash
CLAUDEX_PROFILE=<PROFILE> claudex [PROMPT] [CLAUDE_ARGS...]
```

| 引数 / オプション | 説明 |
|----------|------|
| `CLAUDEX_PROFILE` | プロファイル名、またはスマートルーティングの場合は `auto` |
| `PROMPT` | 非対話（ワンショット）実行のための任意のプロンプト |
| `CLAUDEX_MODEL` | このセッションのデフォルトモデルを上書き |
| `CLAUDEX_HYPERLINKS=on` | OSC 8 ターミナルハイパーリンクを強制有効化 |
| `--print` | レスポンスを出力して終了（非対話モード） |
| `--dangerously-skip-permissions` | すべての権限プロンプトをスキップ（CI/CD で `--print` と併用） |
| `ARGS...` | `claude` に渡す追加引数（`--` の後） |

```bash
# Grok を使用（対話モード）
CLAUDEX_PROFILE=grok claudex

# 特定のモデルで Grok を使用
CLAUDEX_PROFILE=grok CLAUDEX_MODEL=grok-3-mini-beta claudex

# スマートルーティング
CLAUDEX_PROFILE=auto claudex

# ターミナルハイパーリンクを強制有効化
CLAUDEX_PROFILE=grok CLAUDEX_HYPERLINKS=on claudex

# 非対話ワンショット実行
CLAUDEX_PROFILE=grok claudex "Explain this codebase" --print

# CI/CD パイプラインでの使用（プロンプトなし、権限なし）
CLAUDEX_PROFILE=grok claudex "Fix lint errors" --print --dangerously-skip-permissions

# claude に追加引数を渡す
CLAUDEX_PROFILE=grok claudex --verbose
```

### `profile`

プロバイダープロファイルを管理します。

#### `profile list`

```bash
claudex-config profile list
```

ステータス、プロバイダータイプ、モデルを含む設定済みのすべてのプロファイルを一覧表示します。

#### `profile add`

```bash
claudex-config profile add
```

新しいプロファイルを追加する対話型ウィザード。プロバイダー選択、API キー入力（オプションでキーリング保存）、モデル選択、接続テストをガイドします。

#### `profile show`

```bash
claudex-config profile show <NAME>
```

すべての設定フィールドを含む特定プロファイルの詳細を表示します。

#### `profile remove`

```bash
claudex-config profile remove <NAME>
```

設定からプロファイルを削除します。

#### `profile test`

```bash
claudex-config profile test <NAME|all>
```

特定のプロファイルまたはすべてのプロファイルの接続テストを行います。プロバイダーの `/models` エンドポイントに軽量リクエストを送信します。

### `auth`

プロバイダープロファイルの OAuth サブスクリプション認証を管理します。

#### `auth login`

```bash
claudex-config auth login <PROVIDER> [OPTIONS]
```

| オプション | 説明 |
|-----------|------|
| `--profile <NAME>` | 特定のプロファイルに関連付け |
| `--enterprise-url <URL>` | カスタムエンタープライズサーバー URL（セルフホスト GitLab 等） |
| `--headless` | ヘッドレス（デバイスコード）フローを使用、ブラウザ起動をスキップ |

OAuth を使用してプロバイダーにログインします。対応プロバイダー: `claude`、`openai`、`google`、`qwen`、`kimi`、`github`、`gitlab`。

```bash
# OpenAI にログイン（Codex CLI から読み取り）
claudex-config auth login openai --profile codex-sub

# Claude にログイン（~/.claude から読み取り）
claudex-config auth login claude

# Google にログイン
claudex-config auth login google --profile gemini-sub

# エンタープライズ URL で GitLab にログイン
claudex-config auth login gitlab --enterprise-url https://gitlab.mycompany.com

# ヘッドレスモード（ブラウザなし、デバイスコードのみ）
claudex-config auth login github --headless
```

#### `auth status`

```bash
claudex-config auth status [PROVIDER]
```

現在の認証ステータスを表示します。プロバイダー引数なしの場合、設定済みのすべての OAuth プロファイルのステータスを表示します。

```bash
# すべての OAuth プロファイルを確認
claudex-config auth status

# 特定のプロバイダーを確認
claudex-config auth status openai
```

#### `auth refresh`

```bash
claudex-config auth refresh <PROVIDER>
```

プロバイダーの OAuth トークンを手動でリフレッシュします。トークンは期限切れ時にプロキシによっても自動的にリフレッシュされます。

#### `auth logout`

```bash
claudex-config auth logout <PROVIDER>
```

プロバイダーの保存済み OAuth トークンをシステムキーリングから削除します。

### `proxy`

翻訳プロキシサーバーを管理します。

#### `proxy start`

```bash
claudex-config proxy start [OPTIONS]
```

| フラグ | 説明 |
|--------|------|
| `-p, --port PORT` | プロキシポートを上書き（デフォルト: 設定から） |
| `-d, --daemon` | バックグラウンドデーモンとして実行 |

#### `proxy stop`

```bash
claudex-config proxy stop
```

プロキシデーモンを停止します（PID ファイル経由で SIGTERM を送信）。

#### `proxy status`

```bash
claudex-config proxy status
```

プロキシが実行中かどうかとポートを表示します。

### `dashboard`

```bash
claudex-config dashboard
```

リアルタイム監視とクイック起動のための TUI ダッシュボードを起動します。

### `config`

設定ファイルを管理します。

#### `config show`

```bash
claudex-config config show
```

読み込まれた設定ファイルパス、検索順序、現在の設定値を表示します。

#### `config path`

```bash
claudex-config config path
```

解決された設定ファイルパスのみを表示します。

#### `config init`

```bash
claudex-config config init
```

組み込みテンプレートからカレントディレクトリに新しい設定ファイルを作成します。

#### `config recreate`

```bash
claudex-config config recreate
```

`config.example.toml` から設定ファイルを再作成し、既存の設定を上書きします。

#### `config edit`

```bash
claudex-config config edit
```

`$EDITOR`（または `$VISUAL`）で設定ファイルを開きます。

#### `config validate`

```bash
claudex-config config validate
```

設定の構文、プロファイル参照（backup_providers、router rules）、OAuth プロバイダー設定を検証します。エラーと警告を報告します。

#### `config get`

```bash
claudex-config config get <KEY>
```

キーパス（例: `proxy_port`、`profiles.0.name`）で特定の設定値を取得します。

#### `config set`

```bash
claudex-config config set <KEY> <VALUE>
```

キーパスで特定の設定値を設定します。

#### `config export`

```bash
claudex-config config export
```

現在のマージ済み設定を TOML 形式で標準出力にエクスポートします。

### `sets`

コンフィギュレーションセット（ルール、スキル、MCP サーバーのインストール可能なバンドル）を管理します。

#### `sets add`

```bash
claudex-config sets add <PATH>
```

`manifest.json`（または `manifest.yaml`）を含むローカルディレクトリからコンフィギュレーションセットをインストールします。

#### `sets remove`

```bash
claudex-config sets remove <NAME>
```

名前を指定してインストール済みのコンフィギュレーションセットを削除します。

#### `sets list`

```bash
claudex-config sets list
```

インストール済みのすべてのコンフィギュレーションセットをバージョン、説明、コンポーネントの概要とともに一覧表示します。

#### `sets update`

```bash
claudex-config sets update <NAME> <PATH>
```

新しいマニフェストパスから既存のコンフィギュレーションセットを更新します。

#### `sets show`

```bash
claudex-config sets show <NAME>
```

すべてのコンポーネントを含む、インストール済みコンフィギュレーションセットの詳細情報を表示します。

### `update`

```bash
claudex-config update [OPTIONS]
```

| フラグ | 説明 |
|--------|------|
| `--check` | アップデートの確認のみ行い、インストールしない |

`--check` なしの場合は、GitHub Releases から最新バージョンをダウンロードしてインストールします。
