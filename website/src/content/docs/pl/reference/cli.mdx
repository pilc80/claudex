## Uzycie globalne

```
claudex [CLAUDE_ARGS...]
claudex-config [OPTIONS] [COMMAND]
```

`claudex` is the Claude-compatible launcher. `claudex-config` owns setup and management commands. Use `CLAUDEX_PROFILE`, `CLAUDEX_MODEL`, `CLAUDEX_HYPERLINKS`, and `CLAUDEX_CONFIG` for launch-time claudex settings.

| Opcja | Opis |
|-------|------|
| `--config <PATH>` | Nadpisanie sciezki pliku konfiguracyjnego |
| `--version` | Wyswietl wersje |
| `--help` | Wyswietl pomoc |

Po uruchomieniu bez polecenia Claudex uruchamia pulpit TUI, jesli profile sa skonfigurowane, lub wyswietla wiadomosc powitalna, jesli nie istnieja zadne profile.

## Polecenia

### launch

Uruchom Claude Code z okreslonym profilem dostawcy.

```bash
CLAUDEX_PROFILE=<PROFILE> claudex [PROMPT] [CLAUDE_ARGS...]
```

| Argument / Opcja | Opis |
|----------|------|
| `CLAUDEX_PROFILE` | Nazwa profilu lub `auto` dla inteligentnego trasowania |
| `PROMPT` | Opcjonalny monit dla wykonania nieinteraktywnego (jednorazowego) |
| `CLAUDEX_MODEL` | Nadpisanie domyslnego modelu dla tej sesji |
| `CLAUDEX_HYPERLINKS=on` | Wymus wlaczenie hiperlinkow terminala OSC 8 |
| `--print` | Drukuj odpowiedz i wyjdz (tryb nieinteraktywny) |
| `--dangerously-skip-permissions` | Pomin wszystkie monity o uprawnienia (uzyj z `--print` dla CI/CD) |
| `ARGS...` | Dodatkowe argumenty przekazywane do `claude` (po `--`) |

```bash
# Uzyj Grok (interaktywnie)
CLAUDEX_PROFILE=grok claudex

# Uzyj Grok z okreslonym modelem
CLAUDEX_PROFILE=grok CLAUDEX_MODEL=grok-3-mini-beta claudex

# Inteligentne trasowanie
CLAUDEX_PROFILE=auto claudex

# Wymus hiperlinki terminala
CLAUDEX_PROFILE=grok CLAUDEX_HYPERLINKS=on claudex

# Nieinteraktywne jednorazowe wykonanie
CLAUDEX_PROFILE=grok claudex "Explain this codebase" --print

# Uzycie w potoku CI/CD (bez monitow, bez uprawnien)
CLAUDEX_PROFILE=grok claudex "Fix lint errors" --print --dangerously-skip-permissions

# Przekaz dodatkowe argumenty do claude
CLAUDEX_PROFILE=grok claudex --verbose
```

### `profile`

Zarzadzanie profilami dostawcow.

#### `profile list`

```bash
claudex-config profile list
```

Wylistuj wszystkie skonfigurowane profile z ich statusem, typem dostawcy i modelem.

#### `profile add`

```bash
claudex-config profile add
```

Interaktywny kreator do dodawania nowego profilu. Prowadzi przez wybor dostawcy, wprowadzenie klucza API (z opcjonalnym zapisem w lancuchu kluczy), wybor modelu i testowanie lacznosci.

#### `profile show`

```bash
claudex-config profile show <NAME>
```

Pokaz szczegoly dla konkretnego profilu w tym wszystkie pola konfiguracyjne.

#### `profile remove`

```bash
claudex-config profile remove <NAME>
```

Usun profil z konfiguracji.

#### `profile test`

```bash
claudex-config profile test <NAME|all>
```

Przetestuj lacznosc dla konkretnego profilu lub wszystkich profili. Wysyla lekkie zadanie do endpointu `/models` dostawcy.

### `auth`

Zarzadzanie uwierzytelnianiem OAuth przez subskrypcje dla profili dostawcow.

#### `auth login`

```bash
claudex-config auth login <PROVIDER> [OPTIONS]
```

| Opcja | Opis |
|-------|------|
| `--profile <NAME>` | Przypisz do konkretnego profilu |
| `--enterprise-url <URL>` | Niestandardowy URL serwera korporacyjnego (dla samodzielnie hostowanego GitLab itp.) |
| `--headless` | Uzyj przeplywu bezinterfejsowego (device code), pomin uruchamianie przegladarki |

Zaloguj sie do dostawcy przy uzyciu OAuth. Obslugiwani dostawcy: `claude`, `openai`, `google`, `qwen`, `kimi`, `github`, `gitlab`.

```bash
# Zaloguj sie do OpenAI (odczytuje z Codex CLI)
claudex-config auth login openai --profile codex-sub

# Zaloguj sie do Claude (odczytuje z ~/.claude)
claudex-config auth login claude

# Zaloguj sie do Google
claudex-config auth login google --profile gemini-sub

# Zaloguj sie do GitLab z URL korporacyjnym
claudex-config auth login gitlab --enterprise-url https://gitlab.mycompany.com

# Tryb bezinterfejsowy (bez przegladarki, tylko device code)
claudex-config auth login github --headless
```

#### `auth status`

```bash
claudex-config auth status [PROVIDER]
```

Pokaz biezacy status uwierzytelnienia. Bez argumentu dostawcy pokazuje status dla wszystkich skonfigurowanych profili OAuth.

```bash
# Sprawdz wszystkie profile OAuth
claudex-config auth status

# Sprawdz konkretnego dostawce
claudex-config auth status openai
```

#### `auth refresh`

```bash
claudex-config auth refresh <PROVIDER>
```

Recznie odswiez token OAuth dla dostawcy. Tokeny sa rowniez automatycznie odswiezane przez proxy po wygasnieciu.

#### `auth logout`

```bash
claudex-config auth logout <PROVIDER>
```

Usun przechowywane tokeny OAuth dla dostawcy z lancucha kluczy systemowego.

### `proxy`

Zarzadzanie serwerem proxy tlumaczacym.

#### `proxy start`

```bash
claudex-config proxy start [OPTIONS]
```

| Flaga | Opis |
|-------|------|
| `-p, --port PORT` | Nadpisanie portu proxy (domyslnie: z konfiguracji) |
| `-d, --daemon` | Uruchom jako demon w tle |

#### `proxy stop`

```bash
claudex-config proxy stop
```

Zatrzymaj demona proxy (wysyla SIGTERM przez plik PID).

#### `proxy status`

```bash
claudex-config proxy status
```

Pokaz, czy proxy dziala i na ktorym porcie.

### `dashboard`

```bash
claudex-config dashboard
```

Uruchom pulpit TUI do monitorowania w czasie rzeczywistym i szybkiego uruchamiania.

### `config`

Zarzadzanie plikami konfiguracyjnymi.

#### `config show`

```bash
claudex-config config show
```

Wyswietl zaladowana sciezke pliku konfiguracyjnego, kolejnosc wyszukiwania i biezace wartosci konfiguracji.

#### `config path`

```bash
claudex-config config path
```

Wyswietl tylko rozwiazana sciezke pliku konfiguracyjnego.

#### `config init`

```bash
claudex-config config init
```

Utworz nowy plik konfiguracyjny w biezacym katalogu z wbudowanego szablonu.

#### `config recreate`

```bash
claudex-config config recreate
```

Odtworz plik konfiguracyjny z `config.example.toml`, nadpisujac istniejaca konfiguracje.

#### `config edit`

```bash
claudex-config config edit
```

Otworz plik konfiguracyjny w edytorze `$EDITOR` (lub `$VISUAL`).

#### `config validate`

```bash
claudex-config config validate
```

Zwaliduj skladnie konfiguracji, referencje profili (backup_providers, reguly routera) i ustawienia dostawcow OAuth. Raportuje bledy i ostrzezenia.

#### `config get`

```bash
claudex-config config get <KEY>
```

Pobierz konkretna wartosc konfiguracji po sciezce klucza (np. `proxy_port`, `profiles.0.name`).

#### `config set`

```bash
claudex-config config set <KEY> <VALUE>
```

Ustaw konkretna wartosc konfiguracji po sciezce klucza.

#### `config export`

```bash
claudex-config config export
```

Eksportuj biezaca scalone konfiguracje na stdout w formacie TOML.

### `sets`

Zarzadzanie zestawami konfiguracji (instalowalne pakiety regul, umiejetnosci i serwerow MCP).

#### `sets add`

```bash
claudex-config sets add <PATH>
```

Zainstaluj zestaw konfiguracji z lokalnego katalogu zawierajacego `manifest.json` (lub `manifest.yaml`).

#### `sets remove`

```bash
claudex-config sets remove <NAME>
```

Usun zainstalowany zestaw konfiguracji po nazwie.

#### `sets list`

```bash
claudex-config sets list
```

Wylistuj wszystkie zainstalowane zestawy konfiguracji z ich wersja, opisem i podsumowaniem skladnikow.

#### `sets update`

```bash
claudex-config sets update <NAME> <PATH>
```

Zaktualizuj istniejacy zestaw konfiguracji z nowej sciezki manifestu.

#### `sets show`

```bash
claudex-config sets show <NAME>
```

Pokaz szczegolowe informacje o zainstalowanym zestawie konfiguracji, w tym wszystkie skladniki.

### `update`

```bash
claudex-config update [OPTIONS]
```

| Flaga | Opis |
|-------|------|
| `--check` | Tylko sprawdz dostepnosc aktualizacji, nie instaluj |

Bez `--check` pobiera i instaluje najnowsza wersje z GitHub Releases.
